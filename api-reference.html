API Reference
Content routing, purchase plan optimization, and bidding. Use the fields below to try each endpoint.
POST /optimize
Optimize content purchase plan for a query. Returns signals, selected sources, bidding data (bid_ceiling, our_bid per source), and articles to scrape.

customer_id

POST /feedback
Submit outcome feedback for a prior optimization (from event_id in /optimize). Used for learning publisher performance.
event_id *
sources_cited (JSON array)
answer_quality (0–1)
user_rating (1–5)
correction_made
Response

GET /learn
Return learned publisher performance by query cluster (k-anonymity applied). Used to refine routing over time.
